# タスク運用（AI主導の開発ルール）

## 進め方
1. ここにタスクを追記する（例: T-0001 経費CSVインポート）
2. タスクごとに「仕様・入出力・制約・テスト手順」を先に書く
3. AIに「この仕様で最小PRを作って」と依頼
4. PRは小さく（100〜300行目安）。マイグレーションは別PRでも可
5. セキュリティ情報（鍵・パスワード）は Secrets/.env で管理

## AIへの依頼テンプレ
- 目的：
- 入力：
- 出力：
- ビジネスルール/制約：
- 想定テーブル変更：
- エンドポイント（あれば）：
- テスト手順：
- パフォーマンス・セキュリティ観点：

## Gitルール
- `main` は常に安定
- 作業は `feature/T-0001-short-name` ブランチ
- PRには「仕様」「テスト手順」「スクショ」を必ず添付
